Introdução

JavaScript é uma linguagem de script de plataforma cruzada orientada a objetos. É uma linguagem pequena e leve. Dentro de um ambiente host (por exemplo, um navegador da web), o JavaScript pode ser conectado aos objetos de seu ambiente para fornecer controle programático sobre eles.

JavaScript contém uma biblioteca padrão de objetos, como Array, Date e Math, e um conjunto básico de elementos de linguagem, como operadores, estruturas de controle e instruções. O JavaScript básico pode ser estendido para uma variedade de propósitos, complementando-o com objetos adicionais; por exemplo:

  • O JavaScript do lado do cliente estende a linguagem central, fornecendo objetos para controlar um navegador e seu Document Object Model (DOM). Por exemplo, as extensões do lado do cliente permitem que um aplicativo coloque elementos em um formulário HTML e responda a eventos do usuário, como cliques do mouse, entrada de formulário e navegação de página.
  • O JavaScript do lado do servidor estende a linguagem central, fornecendo objetos relevantes para a execução de JavaScript em um servidor. Por exemplo, as extensões do lado do servidor permitem que um aplicativo se comunique com um banco de dados, forneça continuidade de informações de uma chamada para outra do aplicativo ou execute manipulações de arquivo em um servidor.

JavaScript e Java

JavaScript e Java são semelhantes em alguns aspectos, mas fundamentalmente diferentes em alguns outros. A linguagem JavaScript se assemelha a Java, mas não tem tipagem estática e verificação de tipo forte de Java. JavaScript segue a maior parte da sintaxe de expressão Java, convenções de nomenclatura e construções básicas de fluxo de controle, razão pela qual foi renomeado de LiveScript para JavaScript.

Em contraste com o sistema de classes de tempo de compilação do Java construído por declarações, o JavaScript oferece suporte a um sistema de tempo de execução baseado em um pequeno número de tipos de dados que representam valores numéricos, booleanos e de string. JavaScript tem um modelo de objeto baseado em protótipo em vez do modelo de objeto baseado em classe mais comum. O modelo baseado em protótipo fornece herança dinâmica; ou seja, o que é herdado pode variar para objetos individuais. JavaScript também oferece suporte a funções sem quaisquer requisitos declarativos especiais. Funções podem ser propriedades de objetos, executando como métodos vagamente tipados.

JavaScript é uma linguagem de formato muito livre em comparação com Java. Você não precisa declarar todas as variáveis, classes e métodos. Você não precisa se preocupar se os métodos são públicos, privados ou protegidos e não precisa implementar interfaces. Variáveis, parâmetros e tipos de retorno de função não são explicitamente digitados.

Variáveis

Você usa variáveis como nomes simbólicos para valores em sua aplicação. Os nomes das variáveis, chamados de identificadores, obedecem a certas regras.

Um identificador JavaScript deve começar com uma letra, underscore(_) ou um cifrão($); os caracteres subsequentes também podem ser números(0-9). Como o JavaScript diferencia maiúsculas de minúsculas, as letras incluem os caracteres "A" a "Z" (maiúsculas) e os caracteres "a" a "z" (minúsculas).

Você pode usar letras ISO 8859-1 ou Unicode, como å e ü em identificadores. Você também pode usar as sequências de escape Unicode como caracteres em identificadores. Alguns exemplos de nomes legais são Number_hits, temp99 e _name.

Declarando variáveis

Você pode declarar variáveis de três modos

Com a palavra-chave var. Por exemplo

var x = 50

Essa sintaxe pode ser usada tanto para declarar variáveis globais como variáveis locais.

Simplesmente atribuindo a ele um valor. Por exemplo

x = 42

Isso sempre declara uma variável global. Ele gera um aviso estrito de JavaScript. Você não deve usar esta variante.

Com a palavra-chave let. Por exemplo

let y = 12

Essa sintaxe pode ser usada para declarar uma variável local de escopo de bloco. Veja o escopo da variável abaixo.

Escopo de variáveis

Quando você declara uma variável fora de qualquer função, ela é chama de variável global, porque ela está disponivel para qualquer outro código dentro do documento presente. Quando você declara uma variável dentro de uma função, ela é chamada de variável local, porque ela só está disponível dentro daquela função.

JavaScript antes do ECMAScript 2015 não tinha escopo de instrução de bloco; em vez disso, uma variável declarada dentro de um bloco é local para a função (ou escopo global) em que o bloco reside. Por exemplo, o código a seguir registrará 5, porque o escopo de x é a função (ou contexto global) dentro da qual x é declarado, não o bloco, que neste caso é uma instrução if.

if (true) { var x = 5; } console.log(x); // 5

Este comportamento muda, ao usar a declaração let introduzida no ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined

Variáveis globais

Variáveis globais são na verdade propriedades do objeto global. Em páginas web, o objeto global é o window, portanto pode definir e acessar as variáveis globals usando a sintaxe window.variavel.

Consequentemente, você pode acessar variáveis globais declaradas em uma janela ou frame a partir de outra janela ou frame, especificando o nome da janela ou do frame. Por exemplo, se uma variável chamada numeroTelefone for declarada em um documento, você pode se referir a essa variável de um iframe como parent.numeroTelefone

Constantes

Você pode criar uma constante somente-leitura, denominada constante com a palavra-chave const. A sintaxe de um identificador de constante é a mesma de um identificador de uma variável: deve começar com uma letra, underscore ou cifrão e pode conter caracteres alfabéticos, numéricos ou caracteres underscore.

const PI = 3.14;

Uma constante não pode ter seu valor alterado por meio de atribuição ou ser redeclarada durante a execução de um script. Ela deve ser inicializada com um valor.

A regra de escopo para constantes são iguais às das variáveis de escopo do bloco let. Se a palavra-chave const for omitida, o identificador será considerado uma representação de uma variável.

Você não pode declarar uma constante com o mesmo nome de uma função ou variável no mesmo escopo. Por exemplo:

// ISSO VAI CAUSAR UM ERROR
function f() {}; const f = 5;
// ISSO VAI CAUSAR UM ERRO
function f() { const g = 5; var g; //demonstração }

No entanto, atributos do objeto não são protegidos, a instrução a seguir é executada sem problemas.

const MEU_OBJETO = {"chave": "valor"}; MEU_OBJETO.key = "outroValor";

Tipos de dados

O padrão ECMAScript mais recente define sete tipos de dados:

  • Seis tipos de dados são primitivos
    • Boolean. verdadeiro ou falso
    • null. Uma palavra-chave especial denotando um valor nulo. Como JavaScript diferencia maiúsculas e minúsculas, null não é o mesmo que Null, NULL ou qualquer outra variante.
    • undefined. Uma propriedade de nível superior cujo valor é indefinido
    • Number. 42 ou 3.14159.
    • String. "Meu nome é"
    • Symbol (novo em ECMAScript 2015). Um tipo de dados cujas instâncias são exclusivas e imutáveis.
  • e Objetos

Embora esses tipos de dados sejam uma quantidade relativamente pequena, eles permitem que você execute funções úteis com seus aplicativos. Objetos e funções são os outros elementos fundamentais da linguagem. Você pode pensar em objetos como contêineres nomeados para valores e funções como procedimentos que seu aplicativo pode executar.

If else

Use a instrução if para executar uma instrução se uma condição lógica for verdadeira. Use a cláusula opcional else para executar uma instrução se a condição for falsa. Uma declaração if tem a seguinte aparência.

if (condition) { statement_1; } else { statement_2; }

condição pode ser qualquer expressão avaliada como verdadeira ou falsa. Consulte Boolean para obter uma explicação do que é avaliado como verdadeiro e falso. Se a condição for avaliada como verdadeira, a instrução_1 será executada; caso contrário, a instrução_2 é executada. instrução_1 e instrução_2 podem ser qualquer instrução , incluindo outras instruções if aninhadas.

Você também pode compor instruções usando else if para ter várias condições testadas em sequência, como segue: 

                        if (condition_1) { statement_1; } 

                            else if (condition_2) { statement_2;}

                            else if (condition_n) { statement_n; } else { statement_last; }

No caso de multiplas condições, somente a primeira condição lógica avaliada como verdadeira será executada. Para executar multiplas instruções, agrupe-as dentro de um bloco de instruções ({...}). Em geral, é uma boa pratica sempre usar um bloco de instruções, especialmente ao aninhar instruções if:

if (condition) { statement_1_runs_if_condition_is_true;
statement_2_runs_if_condition_is_true; } else {
statement_3_runs_if_condition_is_false;
statement_4_runs_if_condition_is_false; }

É aconselhável não usar atribuições simples em uma expressão condicional, porque a atribuição pode ser confundida com igualdade ao examinar o código. Por exemplo, não use o seguinte código:

if (x = y) { /* statements here */ }

Se você precisar usar uma atribuição em uma expressão condicional, uma prática comum é colocar parênteses adicionais ao redor da atribuição. Por exemplo:

if ((x = y)) { /* statements here */ }

Condição while

Uma instrução while executa suas instruções enquanto uma condição especificada for avaliada como verdadeira. Uma declaração while tem a seguinte aparência.

while (condition) statement

Se a condição se tornar falsa, a instrução dentro do loop para de ser exeutada e o controle passa para a instrução seguinte ao loop.

O teste da condição ocorre antes que a instrução no loop seja executada. Se a condição retornar verdadeira, a instrução é executada e a condição é testada novamente. Se a condição retornar falsa, a execução é interrompida e o controle é passado para a instrução seguinte ao while.

Para executar multiplas instruções, use o bloco de instruções ({...}) para agrupar essas instruções.

Exemplo:

O loop while a seguir itera enquanto n for menor que três:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

Com cada iteração, o loop incrementa n e adiciona esse valor a x. Portanto, x e n assumem os seguintes valores.

  • Após a primeira passagem: n = 1 e x = 1
  • Após a segunda passagem: n = 2 e x = 2
  • Após a terceira passagem: n = 3 e x = 3

Após completar a terceira passagem, a o condição n < 3 não é mais verdadeiro, então o loop termina.

Declaração de funções

Uma definição de função (também chamada de declaração de função ou instrução de função) consiste na palavra-chave da função, seguida por:

  • O nome da função
  • Uma lista de argumentos para a função, colocados entre parênteses e separados por vírgulas.
  • As instruções JavaScript que definem a função, entre colchetes, {}.

Por exemplo, o código seguinte define uma função simples chmada quadrado:

function quadrado(numero) { return numero * numero; }

A função quadrado recebe um parâmetro, chamado numero. A função consiste em uma instrução que diz para retornar o parâmetro da função (isto é, número) multiplicado por ele mesmo. A instrução de retorno especifica o valor retornado pela função.

return number * number;

Parâmetros primitivos (Como em um número) são passados para funções por valor; o valor é passado para a função, mas se a função aletrar o valor do parâmetro, essa alteração não será refletida globalmente ou na função chamada.
